About

I'm a Software Engineer in Test with around five years of experience making software more reliable across both the front-end and back-end. My work covers the full quality lifecycle, from analysing requirements and defining clear test scenarios with precise preconditions, steps, and expected results in Jira, to designing the automation that keeps them running on every change.

On the back-end I work primarily in C# with xUnit, building integration and contract tests that protect endpoint functionality and guarantee API contracts stay intact through development and deployment. I isolate tests from external dependencies using Moq and WireMock, and run scenario-based suites in containerised environments with Podman, real databases, and RabbitMQ. On the front-end I use Playwright with TypeScript for component testing.

Earlier in my career I built a C# and Selenium automation framework from scratch, maintained it for stability and scale, and led the automation team while mentoring new engineers and reviewing their code. I work the full Scrum cycle and rely on Git and GitLab for version control and collaborative merge requests, always aiming to make quality a shared responsibility rather than a final gate.
